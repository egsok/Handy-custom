## What this is

A Tauri command (`benchmark_transcription_file`) inside Handy that runs a single
WAV file through a matrix of (model × condition) combinations to compare
transcription quality (WER / CER / punctuation) across Handy's speech engines.
Invokable from the DevTools console of a live `tauri dev` instance — no UI.

The command was built to answer: *which model, which prompt settings, which
anti-hallucination params give the best results on long-form Russian speech?*

### External: patched `transcribe-rs` fork

**Location:** `D:\dev\transcribe-rs-fork\`
**Base:** transcribe-rs 0.3.8 (copy of `~/.cargo/registry/.../transcribe-rs-0.3.8`)
**Patch:** `src/whisper_cpp/mod.rs` — added two fields to `WhisperInferenceParams`:

- `n_max_text_ctx: Option<i32>` — wired to `whisper-rs::FullParams::set_n_max_text_ctx`
- `entropy_thold: Option<f32>` — wired to `whisper-rs::FullParams::set_entropy_thold`

Both default `None` (preserve whisper.cpp defaults). When `Some`, they cap
cross-segment context (128) and raise entropy threshold (2.8) per OpenWhispr
PR #552 / ggml-org/whisper.cpp#1507.

**Not currently a git repo on its own** — it's just a directory. If the next
agent wants version control for the fork, `git init` inside it and commit.
The Cargo.toml's `[patch.crates-io]` points at a file path, not a git URL.

## Build environment gotcha (Windows-specific, critical)

Windows MSBuild FileTracker (`FTK1011`) hits `MAX_PATH` (260 chars) when
whisper-rs-sys's cmake nested TryCompile dirs push the path length over the
limit. To avoid: **always set `CARGO_TARGET_DIR=D:/h` before `cargo` or
`bun run tauri dev`**. Shorter target path = shorter nested paths = build
works. This was set up earlier in the session; the working `target/` was
built under `D:\h\`.

| Model id | Engine | Conditions (per model) |
|---|---|---|
| `breeze-asr` (Whisper Large v2 fine-tune) | Whisper | 4: prompt × anti-halluc |
| `turbo` (Whisper Turbo) | Whisper | 4: prompt × anti-halluc |

| `medium` (Whisper Medium) | Whisper | 4: prompt × anti-halluc |
| `parakeet-tdt-0.6b-v3` | Parakeet (ONNX) | 1 (prompt doesn't apply, uses VadChunked max=20s) |
| `canary-1b-v2` | Canary (ONNX) | 1 (max_chunk=30s) |
| `gigaam-v3-e2e-ctc` | GigaAM (ONNX, Russian only) | 1 (max_chunk=20s) |

Whisper-total: 16 conditions × 3 runs = 48 timed runs.
Non-Whisper-total: 3 conditions × 3 runs = 9 timed runs.
Plus 7 warmup runs (1 per model).

### Report artifacts

Written next to input WAV:
- `benchmark-results-YYYYMMDD-HHMMSS.json` — full machine-readable record
- `benchmark-results-YYYYMMDD-HHMMSS.md` — human-readable aggregates table,
  per-run table, and full transcripts grouped by condition

Each run record includes the exact `transcription_prompt`, `custom_words`,
`language`, `use_anti_halluc`, `chunk_count`, `max_chunk_secs` that were
active at measurement time — for audit.

## How to resume (cold start for a new agent)

1. **Branch:** `cd /d/dev/Handy && git checkout bench/whisper-matrix`. The
   work lives here. Don't merge to `main` or `feat/clipboard-multiformat`
   without user's call.
2. **Fork presence:** confirm `D:\dev\transcribe-rs-fork\Cargo.toml` exists
   and `src/whisper_cpp/mod.rs` contains `n_max_text_ctx` and `entropy_thold`
   in `WhisperInferenceParams`. If missing, the build will fail on the
   `[patch.crates-io]` line.
3. **Environment:** always `export CARGO_TARGET_DIR=D:/h` (bash) or
   `$env:CARGO_TARGET_DIR="D:\h"` (PowerShell) before cargo/tauri commands.
4. **Build check:** `CARGO_TARGET_DIR=D:/h cargo check` — should finish
   under a minute on warm cache.
5. **Run benchmark:** start `bun run tauri dev` (with the env var set),
   open the running Handy window, `Ctrl+Shift+I` → Console → invoke (see
   example above).
6. **Models must be pre-downloaded.** The command records `model not
   downloaded` as an error but doesn't download. If any of the 7 models
   are missing, download them via Handy's UI first.

## Known gotchas to re-flag

- **Glob-match state:** the command mutates settings live (language,
  prompt, custom_words, whisper_anti_hallucination) and restores them via
  `SettingsGuard::Drop`. If the command is cancelled mid-run, settings may
  remain mutated — user's UI would show e.g. empty prompt. Fix: close and
  reopen Handy, original settings are persisted on disk.
- **Whisper texts non-deterministic:** `texts_identical` in aggregates is
  expected to be `no` for Whisper (beam search + temperature fallback).
  For non-Whisper ONNX engines it should be `yes`. If yes → suspicious.
- **Warmup:** takes a full pass through warmup WAV per model. On slow
  ONNX models this adds real time. Worth keeping — first call otherwise
  pollutes timing.
- **Vulkan via whisper-vulkan:** first use on a given GPU device triggers
  shader compilation (~30s). That's what the warmup absorbs.
- **The `_transcription_manager` parameter in the command signature is
  unused — we fetch it later via `app.state::<Arc<TranscriptionManager>>()`**
  to work around a borrow-checker edge with Tauri's `State<'_, Arc<T>>`
  inside a loop that also mutates settings. Don't "clean this up" without
  testing; it will reintroduce a borrow conflict.
